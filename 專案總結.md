## 二、完整功能清單

### 租客端（LINE Bot）
- 查詢空房（Flex 卡片輪播）
- 關鍵字搜尋：「台中市 沙鹿區 5000-8000」「沙鹿 8000以下」等自然語言
- 預約看房（選房→日期→時段→通知房東）
- 維修回報（選類型→描述→通知房東）
- 我的預約
- 自動記錄用戶（名稱/頭像/狀態）
- 收租提醒（每月 1、5 號自動推播）

### 多房東架構（核心）
- 每個房東獨立：自己的房源、用戶、預約、維修互不干擾
- 每個房東有自己的 LINE 官方帳號（動態 Webhook：/webhook/landlord/房東ID）
- 租客加哪個房東的 Bot，只看到該房東的房源
- 預約/維修通知發給對應房東

### 房東官網（/site/房東ID）
- 自訂官網名稱 + LOGO（後台上傳）
- 獨立搜尋自己的房源
- 底部推薦其他房東房源 → 連到主站

### Bot 內容自訂（每個房東各自設定）
- 主選單標題、副標、按鈕名稱
- 歡迎語、各步驟提示語、預約/維修成功訊息
- Bot 開關（停用時回固定訊息）
- 主選單按鈕個別開關（查空房/預約/維修/我的預約）★最新

### 管理後台（/admin）
- 多角色登入：總管理員（ADMIN_KEY）看全部；房東（各自金鑰）只看自己
- 用戶/預約/維修/房源管理
- 房源 CRUD + 照片上傳（Cloudinary）
- 房東管理（總管理員）：新增房東、發金鑰、停用/啟用
- 每個房東卡片功能：🤖設定Bot、💬Bot文字、📱設定選單、⚙️官網設定、🌐個人官網、🔑重發金鑰
- 圖文選單編輯器（版型/文字/預覽/套用/開關）
- header 連結：總管理員→小蝸主站；房東→個人官網＋小蝸主站

### 網站
- 首頁、房源列表、房源詳情、房東官網
- 「成為房東」聯絡頁（/contact）
- 「租客註冊會員」（/register）
- 表頭：拿掉全部房源，改成為房東＋租客註冊

## 四、關鍵設定（環境變數）

### Render（Bot）
DATABASE_URL（6543 transaction）、DIRECT_URL（5432 session）、
LINE_CHANNEL_ACCESS_TOKEN、LINE_CHANNEL_SECRET、OWNER_LINE_USER_ID、
ADMIN_KEY、CLOUDINARY_*、SITE_URL

### Vercel（網站）
DATABASE_URL（必須 6543 transaction，否則連線數爆）、
NEXTAUTH_URL、NEXTAUTH_SECRET、REVALIDATE_SECRET、CLOUDINARY_*、
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME

## 五、踩過的雷與解法（重要！）

1. **LINE Flex 400 錯誤** → color 要六碼（#aaaaaa 非 #aaa）；text 不能空字串；圖片 URL 須 https
2. **DB max clients 爆** → DATABASE_URL 用 6543 transaction mode（不是 5432）
3. **prisma db push 卡住** → 6543 不支援 push，schema 要加 directUrl（5432）給遷移用
4. **Unknown argument ownerId** → 網站 schema 沒同步；Bot 改 schema 後，網站也要同步（共用 DB）
5. **動態路由 404** → landlordWebhook.js 和新版 index.js 沒推上去
6. **Next.js 路由衝突** → 同層不能有兩個動態路由（[handle] + [id]）；房東官網改用 /site/[id]
7. **解壓出現 {...} 怪資料夾** → zip 用 -D 參數（不存空目錄項目）
8. **schema 同步** → Bot 改完，網站 prisma/schema.prisma 也要更新（移除 directUrl）

## 六、新增房東標準流程

1. 後台 → 房東管理 → 新增房東（取得登入金鑰）
2. 房源管理 → 新增房源，歸屬給該房東
3. 房東管理 → 🤖設定Bot（貼 Channel Secret/Token/通知ID）
4. 複製卡片的 Webhook URL → 貼到房東 LINE 後台 → Verify → 開 Use webhook
5. （選填）⚙️官網設定、💬Bot文字、📱設定選單
